0TL;DR — context arrives in layers

Claude's "knowledge" inside a sui-pilot session isn't a single payload — it's a stack of conditional layers, each loaded at a different moment and from a different source. Some are always present from the first keystroke; others only appear when you type a slash command, dispatch a subagent, or call an LSP tool. Use the toggles below to see which layers are active in different scenarios.

1Always-on @-import

This is the only sui-pilot mechanism that runs without your asking. The user's global ~/.claude/CLAUDE.md contains a single line:

@~/.claude/sui-pilot/agents/sui-pilot-agent.md

When Claude Code starts a session — any session, in any directory — it resolves that @-import and pastes the 71-line agent file into the system prompt. The doc-first rule, the topic→corpus routing table, the Move 2024 upgrade checklist, the SDK 2.0 migration watch, the post-implementation quality flow: all of it is already in context before you type your first character.

Why this matters

Most context-injection mechanisms are opt-in — you trigger them by typing a slash command or by Claude dispatching a subagent. The @-import is opt-out: it bends every interaction toward the bundled docs, even one-off questions like "how do I push into a vector in Move?"

Without the @-import

module x::y {
  public fun push(v: &mut vector<u64>, n: u64) {
    vector::push_back(v, n);
  }
}

Module-block syntax + free function call style. Pre-Move 2024 idiom.

With the @-import

module x::y;

public fun push(v: &mut vector<u64>, n: u64) {
    v.push_back(n);
}

Module-label syntax + method style. Move 2024. The agent file explicitly flags the rewrite at lines 50‑54.

2Slash command /sui-pilot

The plugin registers six slash commands under commands/. Five of them are skills (covered in §4); the sixth, /sui-pilot, is a routing command. Typing it adds the body of commands/sui-pilot.md to the conversation — a short prologue that tells Claude to hand off to the specialized agent and lists the LSP tools available.

The injected body

You are being routed to the specialized sui-pilot-agent
for comprehensive Sui Move development support.

The sui-pilot-agent provides:
- Doc-grounded guidance using bundled Sui, Walrus, Seal,
  and TypeScript SDK documentation

What's distinctive. Unlike the always-on @-import, the command body is conversation context, not system prompt. It enters the visible transcript and is subject to context-window pressure as the chat grows. This is the lightest, most ephemeral injection mechanism the plugin uses.

3The sui-pilot-agent subagent

The most surprising part of the architecture: the same file that gets @-imported into the global system prompt (layer 1) also doubles as a dispatchable subagent system prompt. Its frontmatter declares 10 MCP tools, model opus, and the standard editor/shell toolset:

Source: agents/sui-pilot-agent.md:1‑25.

Two roles, one file

  • As an @-import (§1): the file body is loaded into the main session's system prompt. The frontmatter (tool list, model) is ignored — that's metadata for subagent dispatch only.
  • As a subagent: the file is the system prompt of a fresh subagent context with no memory of the main conversation. The frontmatter selects which tools and model that subagent is given. This is how /move-pr-review spawns 10 independent reviewers and 1 consolidator (see §7).

Sharing the file means the doc-first rule is enforced identically in both worlds — but the state differs sharply: the main session accumulates conversation; each subagent starts blank.

5move-lsp MCP — runtime ground truth

Mechanisms 1–4 inject static content from files. The fifth mechanism is different: a Model Context Protocol server that wraps move-analyzer and exposes 10 tools whose output is computed at call time from the user's current code. The server is declared in .claude-plugin/plugin.json and launches automatically when the plugin loads:

"mcpServers": {
  "move-lsp": {
    "command": "node",
    "args": ["${CLAUDE_PLUGIN_ROOT}/mcp/move-lsp-mcp/dist/index.js"],
    "env": {}
  }
}

Source: .claude-plugin/plugin.json:15‑21.

Click a tool to see a representative response. The green "just injected" badge marks the moment that response would enter Claude's context window in a real session.

 Click a tool

Each tool runs against a real Move file under the cursor. The response is JSON, structured by tool: diagnostics return spans + messages, hover returns markdown, completions return labels + edits, and so on.

Static vs runtime. Without the LSP tools, Claude reasons from docs. With them, Claude reasons from your compiler's actual output on your file. This is the only injection mechanism whose payload is impossible to predict at design time.

6Doc corpora — on-demand depth

695 documentation files ship with the plugin, split across five Mysten Labs sources. None are auto-loaded; the agent reaches for Glob or Grep over the right corpus when it needs primary-source confirmation. The routing table is the lookup key:

TopicCorpusFiles
Move language: syntax, types, abilities, generics, modules, idioms.move-book-docs/143
Sui runtime: objects, transactions, framework, on-chain finance.sui-docs/339
Walrus storage: blobs, Sites, operators, HTTP API.walrus-docs/84
Seal secrets: encryption, key servers, access policies.seal-docs/14
TypeScript SDK: clients, dapp-kit, kiosk, payment-kit, SDK 2.0.ts-sdk-docs/115

Source: agents/sui-pilot-agent.md:33‑41 (routing table); README.md:17‑25 (file counts).

Why no index? The plugin deliberately ships no precomputed index. Earlier iterations tried it; the eval suite found a hook-based matcher pipeline didn't help for a plugin with only five explicitly-named skills. The agent uses Claude's existing search tools (Glob/Grep) over a flat directory of markdown.

Cross-corpus answers

Some questions span layers. Walrus stores blobs on Sui, so a Walrus question may need .sui-docs/ for object semantics. Seal stores secrets governed by Move access policies, so a Seal question may need .move-book-docs/. The agent file explicitly tells the model to cross-reference: "Walrus and Seal build on Sui, so cross-reference .sui-docs/ when an answer spans layers."

7Putting it together — /move-pr-review fan-out

The most complex flow combines all six mechanisms. /move-pr-review dispatches 11 fresh subagents — 10 parallel reviewers + 1 consolidator — each typed as sui-pilot:sui-pilot-agent. Every one of them starts with an empty context, immediately loaded with the agent-file body (mechanisms 1+3), each reviewer's own task prompt (a variant of mechanism 4), the 10 MCP tools (mechanism 5), and access to the five corpora (mechanism 6).

Click any reviewer or consolidator bubble to inspect its initial context.

What's in each subagent's context?

  • System prompt seeded with agents/sui-pilot-agent.md body (doc-first rule, 10 MCP tools)
  • Task-specific reviewer prompt from skills/move-pr-review/references/reviewer_prompt.md
  • The PR scope bundle (changed files, dependency graph)
  • Empty conversation history — no awareness of other reviewers or the main session

Why independent contexts

If all 10 reviewers shared one window, they'd see each other's reasoning and converge — a single reasoner's blind spots would replicate across all "lenses." The skill enforces fresh contexts so that singleton findings (caught by only one reviewer) survive into the consolidator. The consolidator then re-derives every critical/high claim against the source code, downgrading or rejecting unverified ones. The final output is a single self-contained HTML report.

Source for orchestration spec: skills/move-pr-review/SKILL.md:28‑60.
