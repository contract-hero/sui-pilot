    Honest caveat. The harness aggregates only the parent session's token usage. With the concept-map loaded, Claude often dispatched a background subagent to do a whole task — the parent then emitted only a closing line, so the cells flagged subagent (e.g. 34, 177, 248) are not "cheaper," just uncounted. Any cost-per-task claim drawn from tokens.csv as it stands is partially fiction. Task-04 is the cleanest comparison because both arms kept the work in-process — and there the always-loaded map still read ~15% less cache.
    What the windows above show is solid even where the cost numbers aren't: the map costs a fixed ~+4,200 tok from token zero, and in exchange the agent issues fewer on-demand Glob/Grep/Read calls into the corpora. Whether that trade nets out cheaper is exactly what the subagent confound prevents this run from settling.
    Reference — the two regimes, and how to read the inspectors

     Point — the slim routing table

    The always-on @-import is just the 71-line agents/sui-pilot-agent.md preamble: a Doc-First rule and a 6-row topic→corpus routing table. The agent holds a pointer to the docs and reaches into a corpus with Glob/Grep/Read only when a task needs it. Small static footprint; per-task corpus reads light up the pills.

     Concept-map — the 445-line graph, always loaded

    The same file, plus the ## Ecosystem Knowledge Graph section (~445 lines): a relational map of abilities, the object model, authorization patterns, transactions, and the off-chain stack, with 📖 docs: pointers. It was present in the Vercel-port era as sui.md, chunk-injected by hooks; here it is simply always present in the prompt. Big static footprint from token zero; fewer per-task corpus reads because the orientation is already in the window.

     10 move-lsp MCP tools

    Both regimes share the same runtime context source: move-analyzer via the move-lsp MCP server — diagnostics, hover, completions, goto/type definition, references, document symbols, code actions, inlay hints, rename. This is identical across arms; the regimes differ only in the always-on @-import.
