How we're learning to feed Claude the Sui docs

sui-pilot is a doc-first plugin: it ships the current Sui, Move, Walrus, Seal and TypeScript-SDK docs so Claude reads them instead of trusting stale training memory. The hard question was never which docs — it's how they should enter the context window. This is the running log of how we measure that, one experiment at a time.

The story so far — three experiments, one question

  1. 1 First — build a way to measure it

    An eval harness for context, not just code

    You can't tune what you can't measure. We built a 27-task A/B suite spanning Move 2024 syntax, one-time-witness, SDK 2.0, stale-training traps, multi-file refactors and ambiguous specs — scored with literal pass/fail, an LLM rubric, token economics and an optional compile gate. Three committed baselines, with the framing axis spelled out honestly. This is the instrument every later experiment runs on.

  2. 2 Then — see what actually loads

    A lab for the context window itself

    Before changing what we inject, we mapped what already enters Claude's head during a real Sui workflow: bundled docs, MCP tool calls, skill payloads, the agent file — step by step. The interactive tour makes the invisible visible, and it's why we can reason about a doc-first plugin behaving differently from a vanilla model rather than just asserting that it does.

    Open the lab →

    Source: SUI_PILOT_TOUR.html

  3. 3 Now — point at the docs, or index them?

    The point-vs-index experiment

    With the instrument and the lab in place, we ran the experiment we'd been circling: should the agent be handed a pointer to the docs (a 7-row routing table it searches on demand), or should the docs' relational map be indexed straight into the prompt (a 445-line ecosystem graph that's always present)? Same build, byte-identical everything except that one file. The answer wasn't the clean win we expected — and where it got murky turned out to be the most useful part.

What's still open

Experiment 3 is a checkpoint, not a verdict. Code quality came out indistinguishable between the two shapes, and wall-time and cache-reads dropped with the always-loaded graph — but the cost claim isn't trustworthy yet, because the graph nudged the agent into dispatching background subagents whose tokens our harness doesn't count. Until tokens.csv walks the full subagent tree, "cheaper" stays unproven.

So the log continues. The recurring lesson across all three experiments is that measuring what context does is harder than changing it — the methodology is part of the result.
